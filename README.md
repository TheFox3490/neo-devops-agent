# Neo DevOps Agent: AI-ассистент для анализа инцидентов (SRE & DevOps)

## О проекте

**Neo DevOps Agent** — это интеллектуальный агент, разработанный для помощи SRE-инженерам, DevOps-специалистам и разработчикам в анализе эксплуатационных инцидентов (postmortems). 

**Проблема:** При возникновении аварий или при проектировании отказоустойчивых систем инженеры тратят огромное количество часов на поиск релевантного исторического контекста: были ли похожие аварии в прошлом, какие решения реально помогли, а какие действия только усугубили ситуацию.

**Решение:** AI-агент, который имеет доступ к обширной базе знаний постмортемов. Агент способен понимать контекст проблемы, находить релевантные инциденты, объяснять причины (root cause), извлекать выученные уроки (lessons learned) и давать конкретные рекомендации на основе исторического опыта.

---

## Ключевые возможности и сценарии использования (Use Cases)

Агент способен отвечать на сложные технические вопросы, требующие агрегации данных из разных отчетов:
* *"Были ли в нашей практике инциденты, связанные с деградацией базы данных при резких скачках трафика?"*
* *"Какие типичные ошибки допускаются при автомасштабировании Kubernetes-кластеров?"*
* *"Как крупные компании обычно реагировали на memory leaks в production-окружении?"*

Агент самостоятельно принимает решение о том, нужно ли уточнить запрос пользователя, какой формат ответа выбрать (сравнительная таблица, список кейсов или summary) и на чем сделать акцент (таймлайн инцидента, первопричины или способы решения).

## Техническая реализация и Архитектура

Проект строится на современных паттернах разработки LLM-приложений:

* **Паттерн Агента:** Агент функционирует по принципу **ReAct** (Reasoning and Acting), что позволяет ему "рассуждать" над задачей пользователя и пошагово вызывать необходимые инструменты для сбора информации перед формированием финального ответа.
* **Векторное хранилище (Vector DB):** В качестве ядра базы знаний используется **Qdrant**.
* **Продвинутый поиск (Tools):** Агент снабжен набором инструментов (Tool Calling) для гибкого поиска по базе инцидентов:
  * *Семантический поиск* (по смыслу запроса).
  * *Полнотекстовый поиск* (аналог BM25 для точного совпадения технических терминов и логов).
  * *Поиск по категориям и метаданным* (фильтрация по стеку технологий, времени и типам инцидентов).
* **Архитектура RAG:** Поддерживается два вида извлечения контекста:
  1. Классический RAG (поиск по свободному запросу).
  2. **HyDE (Hypothetical Document Embeddings):** При сложных запросах агент сначала генерирует псевдо-документ (ожидаемый ответ) через ToolCalls, и использует его векторное представление для поиска максимально релевантных реальных постмортемов в базе.

## План развития (Roadmap проекта)

В рамках развития сквозного проекта планируется итеративное усложнение логики:

1. **Фаза 1:** Базовая реализация Retrieval-Augmented Generation (RAG) пайплайна с использованием связки Qdrant + LangChain.
2. **Фаза 2:** Оборачивание логики в ReAct-агента с предоставлением ему инструментов (Tools) для различных видов поиска (гибридный, HyDE). Интеграция Conversation Memory.
3. **Фаза 3:** Переход к графовой архитектуре агента (LangGraph) для строгого контроля циклов, обработки ошибок и управления состоянием (State Graph) при сложном ресерче инцидентов.
